# GitHub Copilot Instructions for Scarmonit-Architecture

You are working on the Scarmonit-Architecture monorepo.

## Project Structure
- **web-portal/**: Main website (scarmonit.com). Pure HTML/CSS/JS or React. Deploy: Cloudflare Pages.
- **agent-api/**: Backend API. TypeScript, Cloudflare Workers. Deploy: `wrangler`.
- **desktop-app/**: Desktop Chat App. Electron, React, Node.js.

## Coding Style & Conventions
- **TypeScript**: Use strict typing in `agent-api`.
- **JavaScript**: Use modern ES6+ syntax in `web-portal` and `desktop-app`.
- **Formatting**: 2 spaces indentation, single quotes, no semicolons (Prettier).
- **Comments**: Add JSDoc for complex functions.

## Specific Technology Context
- **Cloudflare Workers**: Use `worker-configuration.d.ts` for bindings. Use `Hono` or native `fetch` handler patterns if applicable.
- **Electron**: Use IPC patterns for main/renderer communication. Secure defaults (context isolation).
- **React**: Functional components with Hooks.

## Behavior
- When asked to refactor, prefer small, atomic changes.
- Always check `package.json` in the specific workspace before suggesting imports.
- If modifying the API, ensure `wrangler.toml` compatibility.
